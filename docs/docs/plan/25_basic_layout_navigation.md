# Issue #25 基本レイアウトとナビゲーション 実行計画

## 全体像

- 目的: AppShell化した共通レイアウトに合わせてヘッダー/フッター/カラー切替を再構成し、詳細ページのUIとマップ表示を安定させる。
- 背景: `feature/#25_basic_layout` ブランチでナビゲーション刷新を進めたが、都道府県詳細マップが描画されない・カード幅が固定化するなどの副作用が確認された。
- スコープ: 共有レイアウトコンポーネント群、自治体/都道府県詳細ページのレイアウト、Mapbox連動コンポーネントの不具合修正とUX改善。

## 進捗状況（チェックリスト）

- [x] AppShell・AppBar・ColorModeProvider を導入し、MUI v7ベースの共通レイアウトを構築した。
- [x] 自治体詳細ページを左右2カラム構成へ移行し、マップ/テーブルの連動を復旧した。
- [x] 自治体・都道府県双方の Mapbox 表示で `ResizeObserver` を組み込み、初期描画・レイアウト変化時の空白を解消した。
- [ ] `/prefectures/[code]` にてクラスタ展開・一覧テーブル連携が期待通り動作するか手動QAを実施する。
- [ ] レイアウト刷新内容をデザイン/ドキュメントへ共有し、スタイル指針のフィードバックを反映する。

## 発見と驚き

- Mapbox コンテナの高さが `min-height` のみだと JavaScript 側でサイズ確定せず、レイアウト切り替え直後に `resize()` が走らないため地図が空白になるケースがあった。
- 自治体詳細では親コンテナの `display: flex` 指定がなく、Mapbox コンテナの `height: 100%` が効かず描画領域が潰れていた。Prefecture側との差分で発覚。
- 左側情報カードの `width: fit-content` 指定はブラウザ間で挙動差が大きく、`clamp` と `flex-basis` の併用でバランスが取りやすいことを確認。

## 決定ログ（日時と理由）

- 2025-10-23 05:05 JST: `BoardsClusterMap` に `ResizeObserver` ベースの `resize()` トリガーとフルハイトレイアウトを追加し、都道府県マップの初期描画を安定化する方針を採用。
- 2025-10-23 05:32 JST: 自治体詳細側の `MunicipalityBoardsMap` にも同等の `ResizeObserver` とコンテナ参照を適用し、マップ非表示問題の恒久対策とする。
- 2025-10-23 06:18 JST: `MunicipalityBoardsMap` のルートコンテナを `flex` で拡張し `MapboxMap` に `flex: 1` を付与、Mapbox ルート要素の高さが 0 になる問題を解消。
