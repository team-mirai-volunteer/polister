## アーキテクチャ原則

1. **関心の分離**: 各レイヤーが明確な責務を持つ
2. **依存性の逆転**: 上位レイヤーが下位レイヤーに依存しない
3. **テスト可能性**: すべてのコンポーネントが単体テスト可能
4. **保守性**: コードの変更が他の部分に影響を与えない
