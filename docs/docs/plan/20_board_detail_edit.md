# Issue #20 掲示場詳細・編集機能 実行計画

## 全体像

### 目的

- 掲示場の詳細情報を閲覧できるページを実装
- 掲示場データの個別編集機能を実装
- 変更履歴を記録・表示する仕組みを構築

### 背景

- 現在、地図上で掲示場の位置情報は表示できるが、詳細情報を確認する画面がない
- データの編集機能がなく、誤りの修正や情報の更新ができない
- データの変更履歴が追跡できず、監査証跡が不十分

### スコープ

**含む:**

- 掲示場詳細画面（`/boards/[id]`）の新規作成
- 掲示場の詳細情報取得ユースケース
- 掲示場編集機能（全フィールド対応）
- 変更履歴の自動記録（BoardHistory）
- 変更履歴の表示UI
- 地図上での位置調整機能

**含まない:**

- ユーザー認証機能（Issue #23で実装予定、現時点では未認証でも編集可能とする）
- 複数掲示場の一括編集
- 掲示場の削除機能（論理削除は別Issue）
- 画像アップロード機能（既存のBoardImageは表示のみ）

## 発見と驚き

### 既存実装の確認結果

- BoardLocationエンティティが既に存在（地図表示用の簡略版）
- BoardHistoryモデルがスキーマに定義済み
- ChangeReason enumに必要な値が揃っている
- PostGISでの空間インデックスが設定済み

### 技術的な課題

- PostGISのgeography型とPrismaの連携方法
- JSON形式でのbeforeData/afterDataの差分表現
- 地図上でのドラッグ＆ドロップUI（Mapbox GL JS）

## 決定ログ（日時と理由）

### 2025-11-09 作成時

- **詳細画面と編集機能を同一Issueで実装**: 密接に関連しており、分離すると重複実装が発生するため
- **認証機能は後回し**: Issue #23で実装予定のため、現時点では認証なしで実装
- **Board集約ルートの作成**: 既存のBoardLocationは地図表示用の軽量版であり、完全なドメインモデルが必要
- **変更履歴は全フィールドをJSONで記録**: 柔軟性を確保し、将来のスキーマ変更に対応
